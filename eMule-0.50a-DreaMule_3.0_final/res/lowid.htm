사용자의 네트웍에 문제가 있어 로우아이디(LowID)가 발생하였습니다.
프루나가 사용하도록 설정된 통신 포트(TCP)에 다른 사용자가 연결을 할 수가 없어 다른 사용자와 자료 교환에 제약이 따릅니다 .
일반적으로 이 문제는 인터넷 공유기(라우터,IP공유기) 또는 방화벽 사용으로 인해 발생하는 문제입니다.
문제 해결 방법
공유기(라우터,IP공유기) 사용 PC에서 사용하는 경우
공유기라는 것은 하나의 인터넷 회선(ADSL,VDSL)에 두대이상의 PC하여 사용할때 사용하는 장비입니다.
공유기는 기본적으로 공유기 설정에서 사용자가 설정한 통신 포트 이외의 모든 통신 포트를 차단을 하게 됩니다.
그렇기 때문에 프루나를 사용하려면 공유기의 설정에서 DMZ호스트설정(모든 포트 개방)을 하거나 프루나가 사용하는 통신 포트 TCP 4662를 포트포워딩(특정 포트 개방)를 개방하는 설정을 해야 합니다.
모든 공유기에는 제조사마다 설정방법에 차이가 있을뿐 위의 설정은 모두 있습니다. 공유기 설명서를 참고하거나 제조사에 문의를 하면 쉽게 할수 있습니다.
방화벽내에서 사용하는 경우
방화벽은 방화벽 설정에 따라 통신 포트를 차단 또는 개방을 합니다.
그렇기 때문에 프루나를 사용하려면 방화벽을 종료하거나 방화벽 설정에서 프루나가 사용하는 통신 포트 TCP 4662를 개방하는 설정을 해야 합니다.
XP에는 자체 방화벽이 있어서 설치시에 기본설정으로 모든 포트를 차단하게 될수도 있기 때문에 사용자가 설정을 꺼주어야 합니다.
회사, 학교등에서 사용하는 경우
회사, 학교등에는 대부분 네트웍에 방화벽이 설치되어 있습니다. 이 방화벽은 사용자가 마음데로 설정을 할수 있는 것이 아니라 네트웍 관리자만이 관리를 하게 됩니다.
이럴 경우에는 대부분 사용이 불가능 하지만 네트웍 관리자에세 개방된 포트를 문의해서 프루나 설정에서 프루나 사용 포트를 개방된 포트로 설정하여 사용하거나, 프루나가 사용하는 포트를 개방해 달라고 요청을 해야합니다.
일시적인 현상일 경우
1. 정보창에서 서버 연결을 끊고 다시 연결을 해봅니다.
2. 해결되지 않을때는, 프루나를 종료한 후 잠시 후에 다시 실행합니다.
3. 그래도 해결되지 않을때는, 프루나를 종료하고 PC 를 껏다가 다시 켠 후 프루나를 실행합니다.
위의 방법을 해도 문제가 계속 발생하는 경우
프루나가 기본적으로 사용하는 네트웍 포트가 막혀 있는 경우에는
프루나 옵션 > 연결 > 고급 설정에 기본 포트(TCP)를 80 이나 다른 값으로 설정 한 후 다시 시작합니다.
